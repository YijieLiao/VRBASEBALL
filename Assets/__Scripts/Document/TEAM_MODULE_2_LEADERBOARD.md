# 模块 2：排行榜信息存储 — 分工汇报材料

## 我负责什么

**一句话**：游戏分数怎么存、怎么排、怎么显示出来，包括结算流程。

---

## 用到了哪些脚本

都在 `Assets/__Scripts/` 下，共 5 个文件：

| 脚本 | 在哪 | 干嘛的 |
|------|------|--------|
| `LeaderboardManager.cs` | `GamePlay/` | 排行榜核心：存取数据到本地文件、排前十、判断有没有进榜 |
| `LeaderboardPanel.cs` | `UI/` | 排行榜显示面板：10 行榜单、哪些字段显示、玩家高亮 |
| `ResultCoordinator.cs` | `GamePlay/` | 结算流程的指挥：判定→输名字→写数据→显示→再来一局 |
| `SettlementTestDriver.cs` | `Test/` | 测试用：一键填假数据、模拟高分低分（只测试场景用） |
| `LeaderboardEntry` | 在 `LeaderboardManager.cs` 里 | 数据模型：一条记录包含名字、动物、分数、日期 |

---

## 我做了什么

### 1. 排行榜存到本地文件

- 存成 JSON 文件，路径是 `derby_leaderboard.json`
- 在 Pico 设备上会存到 App 的私有目录，不会丢
- 游戏关了再开，排行榜数据还在
- 最多存 20 条，超出自动删最末尾的
- 每条存：玩家名、动物名、分数、日期（月-日）

### 2. 分数排序和排名

- 新分数进来，自动按从高到低插入到正确位置
- `IsHighScore(分数, 前几名)` — 判断有没有进前 N 名
- `GetRank(分数)` — 返回当前排第几
- `GetTopScores(10)` — 返回前十名的列表

### 3. 两个排行榜面板

**房间常驻排行榜**：
- 放在房间视角里，一直显示
- 每次玩家进入房间自动刷新
- 没有任何高亮，纯展示
- 没有"再来一局""返回房间"按钮

**结算排行榜**：
- 游戏打完才弹出来
- 如果玩家这局进了前 10，对应那行文字变白色，背景加亮
- 有"再来一局"和"返回房间"按钮
- 带淡入淡出动画

两个面板用的是**同一个脚本** `LeaderboardPanel.cs`，靠一个叫 `Refresh On Room Enter` 的勾选框区分行为。

### 4. 可以选显示哪些列

Inspector 里 5 个勾选框控制：

- 排名 ✅
- 玩家名 ✅
- 动物 ✅
- 分数 ✅
- 日期 ❌（默认不显示）

勾哪个显示哪个，不勾就不拼那一段文字。

### 5. 完整结算流程

游戏打完了 → 看一下分数进没进前 10：
- **没进** → 直接显示排行榜
- **进了** → 弹出名字输入面板 → 玩家用键盘输名字 → 点确认 → 写入排行榜 → 显示排行榜（那行高亮白色）
- "再来一局"重新打，"返回房间"回房间

## 怎么实现的（核心思路）

```
游戏结束，总分 1500 传过来
  ↓
LeaderboardManager.IsHighScore(1500, 前10名)
  ├─ 没进 → 直接显示 LeaderboardPanel，没有高亮
  └─ 进了，排第 3 名
       ↓
     弹出 NameInputPanel（键盘那组的）
       ↓
     玩家输完名字点确认
       ↓
     LeaderboardManager.AddEntry("玩家名", "COW", 1500)
       → JSON 序列化 → 写到本地文件
       ↓
     LeaderboardPanel.Show(3)  ← 告诉面板第 3 行要高亮
       → 第 3 行: 文字白色 + 背景亮
       → 其他行: 默认棕色
```

- `ResultCoordinator` 是指挥中心，把"判定""输入""存储""显示"串起来
- 数据流：`LeaderboardManager`（存储）↔ `ResultCoordinator`（协调）↔ `LeaderboardPanel`（显示）
- 用 JSON 存是因为人类可读、调试方便、Unity 自带支持

## 遇到的坑

**最大的坑**：第二次打游戏时，确认名字后排行榜里出现了两条一模一样的记录。原因是确认按钮被代码和 Inspector 各绑了一次，点了触发两次。删掉代码里的绑定就好了。

另外一个细节：刚开始排行榜在游戏过程中就会出现，后来加了 CanvasGroup 初始隐藏——alpha=0、不可交互、射线穿透，彻底解决。
