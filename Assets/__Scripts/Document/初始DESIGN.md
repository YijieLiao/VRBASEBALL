**《Animal Baseball》集成文档**

**一、最简要概况**

游戏名称：Animal Baseball 动物棒球

游戏目标：以VR游戏的形式，寻求一套适配体验的交互模式，最终能给**单人玩家**呈现棒球的**全局体验**

游戏的视角：**第一人称VR视角与上帝视角切换进行**

平台：Pico VR

开发仓库链接：https://github.com/YijieLiao/VRBASEBALL

**二、概念阐述**

游戏特点：诙谐轻松但大体上还原棒球规则的游戏

画面风格：中低面数卡通风格

游戏角色：以低面数动物作为场上球员，在简化形象达成玩法与画面的协调的同时，减轻团队的美术压力

玩家扮演一位身处**孩童房间**的“玩家”，面前摆放着一套精致的**微缩动物棒球桌游**。游戏的核心体验，是在**现实视角**与**桌游内的微缩世界视角**之间自由切换，完整体验一场趣味十足的动物棒球比赛。

在现实视角下，你以正常尺寸观察桌面，俯瞰整个迷你球场，掌握全局局势。当比赛进行到击球、跑垒等关键环节时，视角会进入桌游内部，让你**化身场上的小动物球员**，以第一人称亲身参与击球、投球等动作。

游戏采用轻松可爱的低多边形卡通风格，以动物角色带来诙谐有趣的氛围。通过**现实客厅**与**微缩球场**的双场景切换，结合 VR 独有的沉浸感，让玩家在 “玩桌游” 和 “成为桌游角色” 之间无缝转换，既保留棒球运动的策略与操作乐趣，又带来充满想象力的新奇体验。

**三、游戏机制与规则**

1\. **对于棒球规则的分析**

完整规则参考：https://www.bilibili.com/video/BV1Fa411P7P7 <-60min

简单叙述棒球规则：

由进攻方一人击球，防守方九人防守

防守方投手丢球，进攻方击球手把球打出去。

击球手在顺利击球之后需要开始跑步，如果经过三个垒包回到本垒就+1分

防守方需要尽力捡到球，如果直接接到球就可以把击球手淘汰。如果没有则需要让球员拿着球接触或者堵住跑步的进攻方将其淘汰。

所以，棒球的可玩性体现在：

投球和击球方面具有**博弈：**防守方如何投球，进攻方是否击球，对于策略的选择提出要求

操作具有精度需求：挥棒击球的准度、方向、力度

追逐：与对手存在纯粹的速度较量

3\. **玩法设计**

1\. **进攻 - 击球阶段**

**视角**：第一人称 VR 视角

**交互**：物理挥棒模拟，系统根据碰撞精度、方向和力度计算球的轨迹 。

**直接判定逻辑**：三振出局（3次好球未中直接重置）、界外球（如果是界外球会尝试给特写，直接宣告界外）、四坏保送（当前击球手直接转场至一垒）

**延迟判定逻辑**：接杀（当前角色出局）、全垒打（会有UI提示，可以继续跑）

**环节内AI**：投球手

\*直接判定逻辑的意思是：在完成击球的动作之后立马进行判定

\*延迟判定逻辑的意思是：玩家在击球之后立马进入跑垒阶段，一边跑会一边进行的判定

2\. **进攻 - 跑垒环节**

**视角**：静态上帝视角，相机位置固定在球场上方

**交互**：原地摆臂驱动加速。通过检测手柄的摇晃频率，转化为跑垒者的位移速度 。

**判定逻辑**：封杀（守方持球先于玩家到达垒包）；触杀（防守方在玩家到垒前，用持球的手触碰玩家身体）

**环节内AI**：地方场员、其他跑垒员

3\. **防守 - 投球阶段**

**视角**：第一人称 VR 视角 。玩家此时“降临”到桌面的投手棋子身上

**交互**：**物理投球模拟**。玩家按住手柄扳机键模拟挥臂投球，释放瞬间的速度与角度决定球的轨迹。

**策略博弈**：

**球种选择**：通过摇杆选择（如直球、变化球），系统微调物理球体的受力参数。

**判定逻辑**：

**好球 (Strike)**：进入好球区且击球手未挥棒或挥棒落空。积累 3 次则触发“三振出局”逻辑。

**坏球 (Ball)**：偏离好球区且击球手未挥棒。积累 4 次触发“四坏保送”转场。

**环节内AI**：击球手

4\. **防守 - 守备阶段**

**视角**：静态上帝视角 。相机升起，玩家以“桌游玩家”身份俯瞰全场

**核心机制**：守备内部再拆分为：接球+传球+追逐两个阶段

接球部分：

最靠近计算落点的AI会被玩家以上帝视角俯身，脚下出现黄圈，此时，玩家会有一个冲刺槽判定，需要在倒计时内达到某一个速度（交互方式：原地摆臂驱动加速），会出现跑过了/接杀/没跑到三种情况

接杀则直接结束该回合

如果跑过了或者没跑到，小动物自动靠近球捡起来，玩家进入传球阶段

传球部分：

所有队友不设AI，玩家将进行有子弹时间的传球，传球次数有一定限制

当玩家确认完成传球之后，玩家可以用某个交互方式确认，接下来进入追逐部分

追逐部分：

左手手柄：导航/位移控制

右手手柄：动力/频率控制

环节内AI：敌方跑垒员

4\. **代码规范**

1\. **事件总线驱动 (Event Bus)**

**核心思想**：采用观察者模式，将“发布者”与“订阅者”解耦 。

**规范要求**：

严禁在非必要情况下让脚本高度耦合（例如 Player 脚本直接引用 UIManager） 。

使用 C# 委托实现全局事件电台 。

**生命周期管理**：在 OnEnable 中订阅事件，必须在 OnDisable 或 OnDestroy 中取消订阅，否则会导致内存泄漏或程序崩溃 。

2\. **游戏生命周期管理**

**核心思想**：通过状态机（State Machine）统一控制游戏的各个阶段（如初始化、菜单、进行中、暂停、结算）

**规范要求**：

由 GameManager 统一管理状态跳转，确保每一时刻游戏只处于一个状态 。

禁止在 Update 中通过 if 轮询判断状态，应利用状态切换时的“回调函数”执行业务逻辑 。

3\. **UI 架构 (MVVM 模式)**

**核心思想**：实现逻辑与表现的分离（Model-View-ViewModel） 。

**规范要求**：

**View**：仅负责显示，不处理核心逻辑 。

**Model**：纯粹的数据存储 。

**ViewModel**：作为胶水层，负责监听数据变化并驱动 UI 更新 。

由 UIManager 统一控制 UI 预制体的加载与生命周期，并设立多个 UI 挂点（如背景、常规、弹窗） 。

4\. **数据管理策略**

**核心思想**：区分静态配置与动态存档 。

**规范要求**：

**ScriptableObject (SO)**：用于存储“出厂设置”等不可变数据.

**JSON**：用于存储运行时的玩家存档数据，便于持久化到硬盘及手动调试查看 。

5\. **单例模式与依赖注入 (DI)**

**核心思想**：规范全局唯一访问点 。

**规范要求**：

仅对确实全局唯一的模块（如 GameManager, UIManager）使用单例 。

为了解耦，推荐使用依赖注入（通过构造函数或框架）来传递对象，避免过度依赖全局变量 。

严禁随意 new 一个本应全局唯一的 Service 实例 。

6\. **实体状态机 (FSM)**

**核心思想**：NPC 或怪物必须使用有限状态机管理行为 。

**规范要求**：

将巡逻、追逐、攻击等行为拆分为独立的类，避免臃肿的 if-else 判断 。

状态切换时应配合事件总线触发视觉或音效反馈 。

7\. **接口驱动设计**

**核心思想**：奉行“少继承，多组合”的原则 。

**规范要求**：

使用接口（Interface）定义通用行为（如 IDamageable），使不同种类的物体能以统一的方式被处理 。

推荐使用 TryGetComponent&lt;T&gt; 性能优化写法来获取并判断接口 。

8\. **异步资源加载 (Addressables)**

**核心思想**：替换传统的 Resources.Load 以避免加载大资源时造成的画面卡顿 。

**规范要求**：

所有资源加载必须采用异步请求方式 。

利用其自动追踪引用的特性进行内存管理 。

9\. **输入系统 (Input System)**

**核心思想**：使用新版 Input System 插件 。

**规范要求**：

禁止直接使用 Input.GetKeyDown 。

通过配置 Action 表定义动作，实现代码与具体按键输入的解耦，方便支持手柄和改键 。

10\. **音频管理 (AudioManager)**

**核心思想**：集中化音频控制 。

**规范要求**：

禁止在每个物体上挂载 AudioSource 。

必须通过 AudioManager 全局单例统一播放 BGM 和音效 。

1\. **发球与微缩物理控制**

分工人员：张潇飞

代码位置：Assets/Scripts/MicroPitcher.cs

是否push完成：是

使用接口阐述：球体对象上挂载 MicroPitcher 脚本

**\[2026-04-13 11 39 32.mp4\]**

**功能说明：** 为微缩桌面场景设计的发球系统。通过纯本地坐标计算与自定义微缩重力，反推出精准的抛物线初速度。确保棒球无论在何种缩放比例下，都能精准地从设定的发球点飞向击球点，并自带防坠落的安全复位机制。

**使用方式：**

确保球体挂载了 Rigidbody 组件，并**务必取消勾选 Use Gravity**（脚本会全面接管物理重力，防止微缩场景下引力过大）。

在同一球体对象上挂载 MicroPitcher 脚本。

运行后：按下 Space（空格键）触发投球，抛物线会自动生成；按下 A 键球体会瞬间切断物理惯性并安全复位到发球点。

**参数调节：**

**修改发球与落点 (Launch/Target Position)：** 填写的是基于父物体的**本地坐标** 。

**调整球速 (Flight Time)：**值**越大**，球在空中的飞行时间越长，球速**越慢**；值**越小**，球速**越快**。

**调整抛物线高度 (Micro Gravity Scale)：**

当你为了让球变慢而调大 Flight Time 时，球飞行的弧度会自动变高

此时必须**同步调小**此参数，以压低飞行弧度，维持微缩桌游真实的视觉感。
