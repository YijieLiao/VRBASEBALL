# Roguelike 模式设计文档

## 设计目标

在纯挥棒击球核心上，注入 roguelike 的乐趣基因：**每次 Run 都不同、选择有代价、高风险高回报、有 meta 成长**。

核心概念：你不是在打棒球比赛——你是一个动物击球手，在越来越疯狂的投球地狱里向上攀登。每一层是新的挑战，每层结束选一项强化，直到三振出局。

---

## 一、Run 结构

```
选角色 → 进入击球区
    │
    ├─ Stage 1 (普通)           → 5球 → 3选1强化
    ├─ Stage 2 (随机类型)       → 5球 → 3选1强化
    ├─ Stage 3 (随机类型)       → 5球 → 商店/强化
    ├─ Stage 4 ...
    │
    └─ 3次三振 → Run 结束 → 结算面板 → Meta奖励
```

### 生命系统
- 初始 **3条命**（三振出局）
- 未击中（挥棒落空）= **-1命**
- 击中（含界外）= **不扣命**
- 0命时 Run 立即结束

### 基础节奏
- 每 Stage **5球**
- 每球都是好球（同经典模式，不区分好坏球）
- Stage 越深，球速基础值递增

---

## 二、Stage 类型

| 类型 | 图标 | 说明 | 出现权重 |
|------|------|------|----------|
| 普通 | 🟢 | 标准发球，当前全部 modifier 生效 | 40% |
| 恶劣天气 | 🟡 | 强制一个负面 modifier（如大雾/强风），但 **得分×2** | 15% |
| Boss | 🟣 | 超快/特殊球路，击中有额外奖励分 | 10% |
| 商店 | 🔵 | 花"跑分"买强化（跑分 = 当前Run累计分） | 15% |
| 休息 | ⚪ | **1球定输赢**，击中回1命，未中不扣命 | 10% |
| 混沌 | 🔴 | 每球随机球路/球速，**3x得分** | 10% |

> Stage 深度 >5 后，恶劣天气和 Boss 权重逐渐上升。

---

## 三、Modifier 系统

每层结束后 **3选1**，选择即时生效，持续到 Run 结束。可叠加（同类型取最高值）。

### 正向强化（Buff）

| 名称 | 效果 | 稀有度 |
|------|------|--------|
| 🧲 磁力球棒 | 击球判定范围 +30% | 普通 |
| 💪 力量训练 | 距离 ×1.3 | 普通 |
| 🦅 鹰眼 | 发球后球短暂"慢放"0.3秒 | 普通 |
| 🌬️ 顺风 | 球飞行时额外+15%前向速度（更容易打远） | 普通 |
| 🎯 精准打击 | 全垒打判定距离线 -1m | 稀有 |
| 🍀 幸运球棒 | 15%概率挥棒落空时自动命中（判为有效落地） | 稀有 |
| 🔥 火焰球棒 | 击中球后计分 ×1.5 | 稀有 |
| 🛡️ 铁壁 | 下一次未击中不扣命（1次消耗品） | 稀有 |
| ⚡ 闪电反应 | 界外球也算有效击中（得分50%） | 史诗 |
| 💎 钻石球棒 | 下一球必为全垒打（1次消耗品） | 史诗 |
| 🌟 全垒打之王 | 全垒打额外+200分 | 史诗 |
| 🔄 再来一瓶 | 所有消耗品类 modifier 次数+1 | 史诗 |

### 诅咒+收益（Trade-off）

诅咒类型的 modifier 一旦选中**不可移除**。

| 名称 | 诅咒 | 收益 |
|------|------|------|
| 🏋️ 重棒 | 挥棒速度 -20% | 距离 ×1.8 |
| 🌫️ 大雾 | 球拖尾/轨迹不可见 | 得分 ×2.5 |
| 🌀 怪风 | 球飞行中随机横向偏移 | 全垒打 +200分奖励 |
| 🎲 赌徒 | 每次击中得分随机 ×0.5 ~ ×3 | — |
| 👁️ 盲区 | 球飞到中段才渲染可见 | 得分 ×3 |
| 🐢 慢动作诅咒 | 你的挥棒速度 -30% | 球速也 -30%，得分 ×1.5 |

## 四、球路多样性

基于现有 `Pitcher.PitchWithSpeed(flightTime)` 和 `microGravityScale` 参数扩展：

| 球路 | 实现方式 | 首次出现 |
|------|----------|----------|
| ⚾ 直球 | 标准抛物线，现有逻辑 | Stage 1 |
| 🌀 曲球 | 发射时加水平初速度 | Stage 3+ |
| 🐌 变速球 | flightTime 先大后小（视觉上慢→突然加速） | Stage 2+ |
| 📉 指叉球 | 飞行末段突然增大 microGravityScale（下坠） | Stage 4+ |
| 🦋 蝴蝶球 | 飞行中随机小扰动 | Stage 5+ |
| 💥 双球 | 前一球落地立刻发下一球，间隔极短 | Boss 专属 |

## 五、风险/回报事件

进入新 Stage 前，有概率触发特殊事件（概率随 Stage 深度递增）。

| 事件 | 选择 A | 选择 B | 选择 C（跳过） |
|------|--------|--------|----------------|
| 🎰 双倍或归零 | 当前得分 ×2 | 得分归零，获得随机史诗Buff | 无事发生 |
| 🩸 血之契约 | 扣1命，获得随机稀有Buff | 扣2命，获得随机史诗Buff | 无事发生 |
| 🎁 神秘盒子 | 接受（50%随机Buff / 50%随机Curse） | — | 无事发生 |
| 🏆 最终一击 | 1球：全垒打→得分×3 | 1球：有效落地→得分×1.5 | 无事发生 |
| 🔮 预知未来 | 预览下个Stage类型+球路 | 直接跳过下个Stage（无奖励） | 无事发生 |

## 六、商店系统

商店 Stage：展示 3 个可购买的 Modifier，用当前 **Run 累计分**购买。

| 稀有度 | 价格 |
|--------|------|
| 普通 Buff | 当前Run分 -15% |
| 稀有 Buff | 当前Run分 -25% |
| 史诗 Buff | 当前Run分 -40% |
| 移除一个 Curse | 当前Run分 -30% |

购买后立即从当前总分扣减，可能掉名次但可能换回更高分。

---

## 七、Meta 成长（Run 间永久）

Run 结束后获得 **🏆 奖杯** = Run总分 / 100（向下取整）。

### 永久解锁项

| 解锁项 | 🏆费用 | 效果 |
|--------|--------|------|
| 额外生命 I | 30 | 开局 4 条命 |
| 额外生命 II | 80 | 开局 5 条命 |
| 好球棒 I | 20 | 开局距离 +10% |
| 好球棒 II | 60 | 开局距离 +20% |
| 重掷令牌 | 30 | 每 Run 强化选择时可重掷 1 次 |
| 幸运护符 | 100 | 每 Run 免死 1 次（0命时触发，回1命） |
| 起手强化 | 50 | 开局自带 1 个随机普通 Buff |
| 奖杯加成 | 40 | 结算奖杯 +20% |

奖杯数据 JSON 持久化，类似 LeaderboardManager。

## 八、Run 结算面板

```
Run结束
  ├─ 显示：到达 Stage 数、总分、最高连击、活跃 Modifier 一览
  ├─ 🏆 获得 XX 奖杯
  ├─ Meta商店入口（花奖杯买永久强化）
  ├─ 再来一局（重新开始Run）
  └─ 返回房间
```

可复用 `SettlementPanel` 的架构，扩展为 Roguelike 版本。

## 九、技术架构

### 新脚本

| 脚本 | 职责 |
|------|------|
| `RoguelikeRunManager.cs` | Run 生命周期：Stage 循环、生命管理、事件触发、对标 ClassicModeRoundManager |
| `ModifierManager.cs` | 当前Run的 Active Modifier 列表管理，提供计分/距离/击球判定钩子 |
| `MetaProgressionManager.cs` | 奖杯货币、永久解锁项，JSON持久化 |
| `RoguelikeHUD.cs` | World Canvas：生命数、Stage数、活跃Modifier图标 |
| `ModifierSelectPanel.cs` | 3选1强化选择面板（World Canvas） |
| `ShopPanel.cs` | 商店面板（World Canvas） |

### ScriptableObject 配置

| SO | 用途 |
|-----|------|
| `ModifierDef` | Modifier 数据定义（名称、图标、效果类型、稀有度、是否诅咒） |
| `StageDef` | Stage 模板（类型、球数、球路配置、出现条件） |

### 现有脚本改动

| 脚本 | 改动 |
|------|------|
| `GameManager.cs` | `GameMode` 枚举加 `Roguelike`，加对应状态处理 |
| `Pitcher.cs` | 新增球路类型接口（曲线球/变速球/蝴蝶球等） |
| `HitJudge.cs` | 可能需要暴露更多判定参数给 Modifier 系统修改 |

## 十、分阶段实施

| 阶段 | 内容 | 能玩出什么乐趣 |
|------|------|---------------|
| **Phase A** | Run循环 + 生命系统 + 球速递增 + 基础结算 | 最基本的 roguelike 节奏：越打越难、死了重来 |
| **Phase B** | 3选1强化 + 8个基础 Modifier | 选择带来的 build 多样性 |
| **Phase C** | Stage 类型（恶劣天气/Boss/休息） + 球路变化 | 每层体验不同，有高潮有喘息 |
| **Phase D** | 风险事件 + 商店 | 高风险高回报的心理博弈 |
| **Phase E** | Meta 成长 + 永久解锁 | "再来一局"的长期动力 |

---

## 十一、与经典模式的关系

- 经典模式不变，作为"快速游玩"入口保留
- Roguelike 模式在房间内作为**第二个模式选择**加入
- 两者共用：Pitcher、HitJudge、ViewTransitionManager、LeaderboardManager
- 排行榜可共用（加 mode 字段区分），或 Roguelike 单开一个榜
